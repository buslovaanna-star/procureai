# ProcureAI — Аналіз закупівель

Streamlit-застосунок для автоматичного аналізу залишків і формування рекомендацій до замовлення.

## Як запустити на streamlit.io (безкоштовно, без Python)

### Крок 1 — Створіть репозиторій на GitHub
1. Зайдіть на [github.com](https://github.com) → **New repository**
2. Назва: `procureai` (або будь-яка)
3. Public → **Create repository**

### Крок 2 — Завантажте файли
Завантажте ці три файли у репозиторій:
- `app.py`
- `requirements.txt`
- `README.md`

Натисніть **Add file → Upload files** і перетягніть всі три.

### Крок 3 — Підключіть до Streamlit Community Cloud
1. Зайдіть на [share.streamlit.io](https://share.streamlit.io)
2. Увійдіть через GitHub
3. **New app** → оберіть ваш репозиторій → `app.py`
4. **Deploy** → через 1-2 хвилини застосунок доступний за посиланням

### Крок 4 — Використання
1. Відкрийте посилання застосунку
2. Завантажте ваш Excel-файл
3. Налаштуйте параметри у боковій панелі
4. Завантажте Excel-звіт з результатами

---

## Структура Excel-файлу

| Вкладка | Зміст |
|---|---|
| Продажі | SKU + пари колонок місяць_кількість / місяць_рентаб% |
| Наявність | SKU + дні на складі по місяцях (0–31) |
| Залишки | SKU + залишок (шт) + в дорозі (шт) |
| Ціни постачальника | Артикул, Цена, Скидка, Наличие, Продано за 30 дн |

## Параметри аналізу

| Параметр | За замовчуванням | Опис |
|---|---|---|
| Мінімальна маржа | 35% | Місяці нижче → виключаються |
| Lead time | 14 днів | Час доставки від замовлення |
| Safety stock | 30 днів | Страховий запас |
| Safety при знижці | 60 днів | Запас якщо знижка ≥ порогу |
| Мін. місяців | 6 | Мінімум чистих місяців для "постійного попиту" |
| Мін. продано | 12 шт | За весь період |
| avail_K α | 0.7 | Коефіцієнт впливу наявності |
| λ | 0.25 | Сила часового зважування |
